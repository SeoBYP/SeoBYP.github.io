프로젝트 설명

Unity3D 엔진을 기반으로 개발된 방치형 3D RPG게임입니다. 사용자가 직접 게임을 플레이하지 않아도 캐릭터가 자동으로 돈을 벌고, 플레이어는 전투, 장비 수집, 스킬 강화 등을 통해서 캐릭터를 육성하며, 다양한 전투 및 성장 콘텐츠를 제공하는프로젝트입니다.

주요 업무

1. 서버 연동 및 관리

  • - Unity Gaming Services(UGS)를 사용하여 서버와 클라이언트 간의 실시간 데이터 동기화 시스템 구축
  • - UniTask와 Reactive Extensions 프레임워크 R3를 활용해 게임 진행 데이터를 실시간으로 수집하고 서버와 동기화하도록 설계

2, 전투 시스템 설계 및 구현

  • - FSM(유한 상태 머신) 디자인 패턴을 적용해 플레이어와 몬스터의 상태(State), 행동(Action), 결정(Decision)을 관리하여 유연한 전투 흐름을 구현

3, 성장 시스템 구현

  • - 플레이어 캐릭터의 레벨업, 스킬 강화, 능력치 증가 등 다양한 성장 요소를 설계하여 성장의 변화를 체감할 수 있도록 강화된 연출을 적용
  • - Reactive Extension 프레임워크 R3를 활용하여 성장 시스템을 구축, 아이템 장착 및 강화, 유물 강화, 펫 장착 및 강화 등 각 콘텐츠에서 스탯 변동에 실시간으로 반응해 스탯이 자동 갱신되고 전투에 반영되도록 설계

4, 성장 콘텐츠 구현

  • - ScriptableObject를 활용한 데이터 테이블 구축과 Google Drive 연동을 통해 기획자가 데이터 테이블을 수정하면 자동으로 반영되도록 설계
  • - Event Bus 디자인 패턴을 사용해 데이터 변화 및 UI 상호작용 시 다른 UI와 시스템에 이를 자동 반영해 데이터 일관성 강화
  • - 인벤토리, 스킬, 유물, 펫, 던전, 버프(아이템, 스킬, 광고, 펫), 미션 등 다양한 성장 콘텐츠를 개발하여 플레이어의 성장 경험을 심화

5, 결제 시스템 구현

  • - Unity In-App Purchase(IAP)를 사용하여 게임 내 상점 결제 시스템 구축
  • - 사용자가 아이템을 구매시 Unity Gaming Services에 구매 로그와 영수증과 결제 내역을 유저 DB에 저장하여 관리

6, 가챠 시스템 구현

  • - ScriptableObject를 활용하여 아이템 소환 확률, 유물 소환 확률, 랜덤 상자 보상 아이템 확률을 연동해 가챠 시스템을 구현
  • - 가중치 랜덤 뽑기 방식을 사용하여, 아이템 타입을 제네릭으로 지정해 객체화하고, 중복 아이템이 없도록 딕셔너리를 활용하여 관리

7, 전투 스킬 구현

  • - 스킬이 강화될수록 더욱 화려해지는 스킬 연출을 통해 플레이어 성장의 체감도를 향상
  • - 킬 사용 시 애니메이션과 싱크를 맞춰 전투의 몰입감 향상

8, 인게임 연출 구현

  • - Unity의 Particle Image 에셋을 활용하여 UI 연출 구현
  • - 아이템 강화 성공 및 실패, 보상 획득, 아이템 소환 연출 등을 제작
  • - 전투 상황에서 보스 몬스터 등장, 몬스터 피격, 몬스터 스폰, 골드 던전 및 강화석 던전 등 인게임 연출 요소들을 구현

9, 데이터 분석 대시보드 구축

  • - Unity Analytics를 활용해서 유저 인 게임 이벤트를 구현 및 수집
  • - 인 게임 이벤트 데이터를 활용하여 리텐션율, 유저 행동 분석 대시보드를 구축해 실시간 데이터 모니터링 및 인사이트 제공

10, CI/CD 파이프라인 자동화

  • - GitHub Actions를 활용해 CI/CD 파이프라인을 구축하여 빌드 및 테스트 자동화
  • - 로컬 빌드 머신과 연동하여 별도의 비용 없이 자동으로 빌드, 테스트, 배포까지 자동화

주요 성과

1, 성능 최적화

ZString을 사용해 문자열 처리 효율을 높여 메모리 사용량을 줄였고, LitMotion으로 기존 DOTween을 대체하여 GC 사용량을 최적화함으로써 안정적으로 60FPS를 나오도록 하여 원활한 게임 플레이 환경을 제공했습니다.

2, 안정적인 서버-클라이언트 데이터 연동

실시간 데이터 동기화와 성능 최적화를 통해 안정적인 플레이 환경을 구축했습니다.

3, 확장성 높은 전투 및 성장 시스템 구현:

추가 콘텐츠 개발이 용이한 구조로 전투와 성장 시스템을 설계해 지속적인 업데이트와 기능 확장을 지원했습니다.

4, 데이터 기반의 게임 밸런스 조정 가능:

Unity Analytics 기반의 데이터 수집과 분석을 통해 유저 행동을 모니터링하고, 이를 바탕으로 게임 밸런스를 세밀하게 조정할 수 있도록 했습니다.

어려웠던 점

1, Unity Gaming Services(UGS)의 데이터 로그 부족 문제:

유저의 게임 진행 상황을 효과적으로 추적하기 어려웠고, PITR(Point in Time Recovery)을 지원하지 않아 유저 데이터 손실 시 별도의 복구 방안이 필요했습니다.

개선한 점

1, AWS DynamoDB 연동:

DynamoDB를 통해 유저 데이터 로그를 실시간으로 추적하여 게임 진행 상황을 기록하고, 데이터 복구 기능을 강화했습니다.

2, Google Drive API 연동:

유저 세이브 데이터를 Google Drive에 안전하게 저장하여 데이터 손실 위험을 최소화했습니다.

3, Unity Analytics 기반 대시보드 구축:

Unity Analytics를 통해 수집한 유저 데이터를 바탕으로 인사이트를 제공하는 대시보드를 구축하여 데이터 기반의 의사 결정을 지원하고, 게임 밸런스 조정에 활용했습니다.
